Capturing Continuity in Software Processes





Data as streams

Software development is more than coding, it's a continous and a complex process with a fast time-to-market that requires monitoring maintainbility, risks and social activities in real-time to stay in the game. We want to enable unconventional data sources to react to events - be it from DevOps data, Reviews from Google Play Store or Stack Traces!



Streaming Processor

Streaming Engines are increasingly becoming popular for unbounded data processing, we intend to develop abstractions on-top of Streaming Engines to include "batteries included" for Software Data Science and Analytics, this includes performing live static analysis of source code, extracting requirements from user reviews or performing sentiment analysis



Query Aggregation

Unifying hundreds of data sources at the query interface requires more than simply joining data streams, we intend to semantically annotate data streams to provide a "natural" way of integrating streams, for instance what fields of a Github Stream can I merge with on a Twitter Stream?



Live Feed

Executing a query should be fired once and updated forever such that developers or other stakeholdes can react to changes - as a core feedback loop!
